Practice Exercise: Introduction to Processes and Process Attributes

Objectives

  • Understand the concept of processes in Linux.
  • Learn how to manage processes using various Linux commands.
  • Gain hands-on experience with process attributes and resource monitoring.

Scenario

As a Linux enthusiast, it's essential to comprehend the fundamentals of processes and how to interact with them effectively. This lab exercise will guide you through understanding, managing, and monitoring processes on a Linux system.

Tasks

Task 1: Viewing Running Processes

  • Open a terminal on your Linux system.
  • Use the ps command to display a list of currently running processes.
  • Pay attention to the columns that show the process ID (PID), the name or command associated with each process, and the status.
  • Look for the process with the PID of 1 and take note of the process.(e.g. /sbin/launchd)

Task 2: Identifying Parent and Child Processes

  • Identify a process from the list generated in Task 1 and note its PID.
  • Use the pgrep command to find child processes associated with the selected process.
  • Verify that the child processes have a different PID but the same parent PID as the selected process.
  • Can you share what command you ran?

Task 3: Viewing Detailed Process Information

  • Choose a process from the list generated in Task 1 and note its PID.
  • Use the ps command with the -p option followed by the PID (e.g., ps -p 12345) to display detailed information about that specific process.
  • Observe the various attributes, including the process status, CPU and memory usage, and more. Take note that you will need to add another parameter and value to show the resource usage

Task 4: Monitoring Process Resource Usage

  • Use the top command to open the real-time system monitoring tool.
  • Observe the list of processes, their resource usage (CPU and memory), and the overall system statistics.
  • Pay attention to the dynamic updating of data and the interactive features of the top command.

Task 5: Sending Signals to Processes

  • Identify a process from the list generated in Task 1 and note its PID.
  • Use the kill command to send a termination signal (SIGTERM) to the selected process (e.g., kill -15 <PID>).
  • Confirm that the process has been terminated.
  • Say run sleep 3600& and then kill it afterwards

Task 6: Background and Foreground Processes

  • Launch a simple command in the foreground (e.g., ping www.google.com) that continuously runs in the terminal.
  • Suspend the foreground process by pressing Ctrl + Z.
  • Use the bg command to send the suspended process to the background.
  • Verify that the process is running in the background using the jobs command.
  • Can you run a command that will kill the ping job

Task 7: Managing Process Priorities

  • Use the nice command to launch a new process with a different priority level (e.g., nice -n 10 command).
  • Observe how the process priority affects its resource allocation.
  • Experiment with different priority levels to see their impact.
  • Can you give at least one way to look for the niceness value of a process?

Conclusion

This lab exercise has provided you with practical experience in understanding and working with processes in a Linux environment. You've learned how to view running processes, identify parent and child processes, inspect process attributes, monitor resource usage, send signals to processes, manage background and foreground processes, and adjust process priorities. These skills are crucial for effective Linux system administration and troubleshooting.
